## 商业定位

一句话：给中文独立开发者、AI服务商、内容运营者提供“每天可执行的AI赚钱机会卡”。

核心差异：

- 不做新闻搬运；每条机会必须含目标客户、收费方式、首个获客动作、交付清单、风险。
- 冷启动用人工收款 + 邮件交付，不等支付系统接入。
- 价格从¥29/月起，降低首次付费门槛；用¥499定制报告拉高客单价。

## 订阅入口和交付链路

1. 用户访问销售页。
2. 点击“立即订阅 / 索取付款码”邮件链接。
3. 运营人员回复付款码或正式支付链接。
4. 用户付款后记录到 `metrics/payment_delivery_tracker.csv` 或客户表。
5. 发送样例包和首期内容：`reports/sample_pack/dist/ce1989c2_revenue_sample_pack_complete.md`。
6. 第7天发送续订/升级提示。

## 获客平台计划（至少3个平台）

### 1. 知乎

- 内容形式：长文 + 回答
- 主题：`2026年AI副业还值得做吗？我用机会卡筛掉90%的伪需求`
- CTA：评论“雷达”领取样例；私信导向销售页
- KPI：7天内 3,000 曝光，60 点击，10 线索，2 付款

### 2. 小红书

- 内容形式：3张图文卡 + 1条案例帖
- 主题：`一个AI自动化项目如何从0报价¥499`
- CTA：主页链接/私信“AI商机”
- KPI：7天内 5,000 曝光，100 点击，20 私信，3 付款

### 3. 即刻 / 微信群 / 独立开发者社区

- 内容形式：挑战帖 + 样例分享
- 主题：`7天测试一个AI收费小项目，我每天发一张机会卡`
- CTA：加入试读名单；¥29早鸟名额20个
- KPI：7天内 80互动，30线索，5付款

### 4. X/Twitter / Hacker News（备用英文验证）

- 内容形式：build-in-public thread
- 主题：`I curated 10 AI micro-SaaS opportunities with pricing and GTM scripts`
- CTA：Substack/Gumroad waitlist
- KPI：50邮箱，验证英文需求；若中文CAC过高则切英文路线。

## 首批宣传文案

### 知乎开头

我最近把AI赚钱项目按“能否在7天内验证收费”重做了一遍筛选，发现大多数所谓机会只有趋势，没有客户、价格和获客路径。
所以我做了一个AI商机雷达：每天只保留3-5条能测试的机会，并写清楚目标客户、报价、交付清单、第一条获客文案和风险。
免费样例放在这里：<销售页URL>。

### 小红书标题

- `别再收藏AI工具清单了，直接测这3个收费场景`
- `我用一张机会卡判断AI副业能不能卖钱`
- `7天AI变现挑战：第1天，给跨境卖家做SEO描述包`

### 社群短帖

我在做一个“AI商机雷达”付费订阅，目标不是资讯，而是每天一张可执行赚钱机会卡：找谁卖、卖多少钱、怎么交付、第一条文案怎么写。现在招20个¥29/月早鸟用户，先送一份样例报告。想看样例回复“雷达”。

## 广告投放前置条件

付费广告只在以下条件全部满足后启动：

1. 自然流量至少产生50个线索或3个付款用户。
2. 销售页访问到邮件/付款点击转化率 >= 3%。
3. 试读到付款转化率 >= 5%。
4. 单用户毛利覆盖至少3倍CAC：¥29产品 CAC<=¥9，¥99产品 CAC<=¥33。
5. 已完成退款规则和客服SOP，避免知识付费平台投诉。
6. 有连续7天内容交付样例，避免广告带来用户后无法履约。

## 每日记录方式

每天把渠道数据写入 `metrics/launch_channels.csv`：日期、平台、链接、曝光、点击、线索、付款、收入、CAC、状态、下一步。

## 7天后决策标准

- Go：付费用户 >= 3 或 线索 >= 50 且点击转化 >= 3%。继续专业版和定制报告。
- Pivot-Go：线索充足但付款少，改成更垂直主题，如“AI自动化服务商获客雷达”。
- No-Go：曝光>5000但线索<10，暂停订阅，改卖一次性模板包。
