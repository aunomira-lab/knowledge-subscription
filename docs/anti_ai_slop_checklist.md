# Substack 反 AI 味内容检查清单

生成时间：2026-05-20 07:00
任务：Substack：重做付费验证与反AI味内容策略

## 1. 核心判断
Substack/知识订阅不能把“AI 草稿数量”当进展。每篇内容必须回答：为什么读者愿意为这篇付费，而不是直接问 AI？

## 2. 发布前检查
- 是否有真实案例、真实选择、真实失败或取舍，而不是泛泛建议？
- 是否有明确目标读者和使用场景？
- 是否提供可执行模板、检查表、数据源或决策树？
- 是否写出“停止条件”和风险，而不是一味鼓励？
- 是否删除了 AI 常见套话："在当今时代"、"总而言之"、"赋能"、"降本增效" 的空泛堆叠？
- 是否有一句话承诺：读者看完能节省什么时间、避免什么坑、完成什么动作？

## 3. 付费样稿验收标准
- 免费帖：讲清一个大众问题，给出 1 个可照做方法，结尾自然引导订阅。
- 付费帖：必须比免费帖多出数据表、模板、复盘、案例拆解或执行包。
- 付费价值门槛：读者愿意为“省 2 小时筛选/少踩一次坑/拿到可复制模板”付 $5-$9/月。

## 4. 编辑流程
1. AI 生成初稿。
2. 人工加入个人判断、真实例子、反面案例。
3. 用本清单逐条打分；低于 7/10 不发布。
4. 发布后记录打开率、点击、回复、订阅、退订。
