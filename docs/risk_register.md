## 一风险评级标准

| 等级 | 含义 | 对应行动 |
|------|------|----------|
| **严重 (Critical)** | 可导致业务中断/重大资产损失 | 必须有备选方案，否则不可执行 |
| **高 (High)** | 可能造成显著影响 | 需要主动风控策略，并监控预警指标 |
| **中 (Medium)** | 有一定影响，可控 | 标准操作规范，定期复查 |
| **低 (Low)** | 影响较小或可忽略 | 标准操作即可 |

## 二Substack 风险评估

### 2.1 风险总览

| 风险项 | 等级 | 概率 | 影响 | 证据来源 |
|---------|------|------|------|----------|
| 地域访问限制 | **高** | 中 | 中国用户可能无法访问或订阅 | 2022年Substack曾在中国被屏蔽 |
| Stripe KYC审核失败 | **高** | 中 | 无法开通付费订阅 | Stripe服务条款限制 |
| 平台政策变更 | **中** | 低 | 抽成比例调整或功能限制 | Substack 2024年曾调整AI内容政策 |
| 内容审核删除 | **中** | 低 | 单篇文章被删或帐号限流 | 社区案例报告 |
| 服务中断 | **低** | 低 | 临时无法发布 | Substack SLA 99.9% |

### 2.2 验证命令/检查清单

```bash
# 验证 Substack 访问可用性
# 1. 从中国大陆网络访问 substack.com
# 检查点: 是否需要VPN才能访问

curl -I https://substack.com
# 期望: HTTP 200 (200)

# 2. 验证 Stripe 账户可用性

# 参考: https://stripe.com/global

# 3. 验证中国用户支付能力
# 检查点: 中国发行的信用卡是否能在Stripe成功扣款
# 测试方法: 创建 $1 测试订阅
```

| 风险 | 缓解措施 | 责任人 | 截止日期 |
|------|----------|----------|----------|
| 地域访问限制 | 同时运营即刻 / 小报童作为备选渠道 | 产品 | 2026-05-18 |
| Stripe KYC失败 | 提前完成Stripe账户验证，准备小报童备选 | 运营 | 2026-05-15 |
| 平台政策变更 | 定期关注Substack公告邮件 | 所有人 | 持续 |

### 2.4 Substack 费用结构确认

```
费用说明 (来源: Stripe customer story + Substack Help Center):
- Substack 平台抽成: 付费收入的 10%
- Stripe 支付处理费: 约 2.9% + $0.30/笔
- 实际到账: 付费金额 × (1 - 0.10 - 0.029) - $0.30

示例计算 ($9/月订阅):
- 毛收入: $9.00
- Substack 10%: -$0.90
- Stripe 2.9% + $0.30: -$0.56
- 净收入: ~$7.54 (83.8%)
```

## 三Stripe 风险评估

### 3.1 风险总览

| 风险项 | 等级 | 概率 | 影响 | 证据来源 |
|---------|------|------|------|----------|
| 中国大陆创作者KYC受限 | **严重** | 高 | 无法完成身份验证，业务无法启动 | Stripe Terms of Service |
| 中国用户支付失败 | **高** | 高 | 目标用户无法完成付费 | 支付行业报告 |
| 账户冻结 | **高** | 中 | 资金无法提现 | Stripe风控案例 |
| 退款/拒付率高 | **中** | 中 | 手续费增加，账号受限 | 业务数据 |
| 汇率风险 | **低** | 中 | 收入波动 | USD/CNY汇率波动 |

### 3.2 验证命令/检查清单

```bash
# 验证 Stripe 支持列表
# 1. 检查创作者所在国家/地区是否在Stripe支持列表
# 参考: https://stripe.com/global
# 关键检查点: 是否支持 "Individual" 类型的商家账户

# 2. 验证中国信用卡支持
# 检查点: 中国发行的 Visa/Mastercard 是否可以在Stripe网关成功扣款
# 测试方式: 使用测试卡号 4242 4242 4242 4242 创建测试订阅

# 3. 验证提现路径
# 检查点: 银行账户所在国家是否支持Stripe Connect payout
# 需要准备: 海外银行账户或Wise/派安盈等代收方案
```

### 3.3 Stripe 地域限制详情

```
根据 Stripe Global 公开信息:
- 中国大陆未在 Stripe 支持的商家注册国家/地区列表
- 香港、新加坡、日本、美国等在支持列表

对于中国大陆创作者的常见方案:
1. 使用海外公司主体注册 Stripe (法律结构复杂)
2. 使用香港个人身份注册 Stripe (需要香港身份证/护照)
3. 转向国内代替方案 (小报童、知识星球)
```

### 3.4 执行风控策略

| 风险 | 缓解措施 | 责任人 | 截止日期 |
|------|----------|----------|----------|
| 中国大陆KYC受限 | **主路径**: 使用支持地区身份注册; **备选**: 小报童快速上线 | 运营 | 2026-05-15 |
| 中国用户支付失败 | 同时提供小报童链接作为备选 | 产品 | 2026-05-18 |
| 账户冻结 | 严格遵守Stripe服务条款，避免高拒付率 | 运营 | 持续 |
| 汇率风险 | 接受USD定价，使用年付降低汇率影响 | 运营 | 持续 |

## 四短视频平台风险评估 (抖音/视频号/快手)

### 4.1 风险总览

| 风险项 | 等级 | 概率 | 影响 | 证据来源 |
|---------|------|------|------|----------|
| 帐号封禁 | **高** | 中 | 内容无法发布，粉丝流失 | 平台公告/社区案例 |
| 内容限流 | **中** | 高 | 曝光量下降，获客效率降低 | 算法调整 |
| 导流限制 | **中** | 高 | 无法在视频/简介中放外部链接 | 平台规则 |
| 评论区封控 | **中** | 中 | 用户无法通过评论引流 | 平台审核 |
| 审核时长延迟 | **低** | 中 | 内容发布不及时 | 算法审核 |

### 4.2 短视频平台引流约束详情

```
各平台导流限制 (来源: 各平台官方规则):

1. 抖音:
   - 个人简介: 可放外部链接 (需达一定粉丝数/认证)
   - 视频内: 禁止显示二维码、其他平台名称
   - 评论区: 敏感词会被自动屏蔽

2. 微信视频号:
   - 个人简介: 可放公众号/企业微信/外部链接
   - 视频内: 可展示公众号名称
   - 引流优势: 和微信生态无缝衔接

3. 快手:
   - 个人简介: 可放外部链接
   - 视频内: 限制较严
   - 评论区: 相对宽松
```

### 4.3 执行风控策略

| 风险 | 缓解措施 | 责任人 | 截止日期 |
|------|----------|----------|----------|
| 帐号封禁 | 严格遵守内容合规清单，避免违禁词 | 运营 | 每篇发布前 |
| 内容限流 | 不依赖单一平台，分散多平台发布 | 运营 | 持续 |
| 导流限制 | 使用口令/暗号方式引导私信 | 运营 | 持续 |
| 评论区封控 | 使用通过审核的引导话术 | 运营 | 持续 |

## 五微信私域风险评估 (公众号/企业微信/个人号/群)

### 5.1 风险总览

| 风险项 | 等级 | 概率 | 影响 | 证据来源 |
|---------|------|------|------|----------|
| 公众号封禁 | **严重** | 中 | 主要获客渠道中断 | 微信公众平台运营规范 |
| 群聊封禁 | **高** | 中 | 用户社群消失 | 微信社区规范 |
| 个人号限制 | **高** | 高 | 无法添加好友/发送消息 | 风控系统 |
| 企业微信审核 | **中** | 中 | 外部联系人无法添加 | 企业微信规则 |
| 支付功能限制 | **中** | 中 | 收款码被限制 | 微信支付规范 |

### 5.2 微信生态封禁触发因素

```
公众号封禁高风险行为 (来源: 微信公众平台运营规范):
- 发布违法违规内容 (资讯类需要ICP备案)
- 过度营销/扰民 (频繁推送广告)
- 诱导分享/流量欺诈
- 使用第三方插件/外链跳转
- 用户举报率过高

群聊封禁触发因素:
- 群内发布违规内容
- 群成员举报
- 风控系统误判
- 主动解散群聊

个人号限制触发因素:
- 短时间内大量添加好友
- 被多人举报
- 发送广告/垃圾信息
- 使用外挂/助手
```

### 5.3 执行风控策略

| 风险 | 缓解措施 | 责任人 | 截止日期 |
|------|----------|----------|----------|
| 公众号封禁 | 进行ICP备案，遵守发布频率规范，平台导流不作为售后渠道 | 运营 | 2026-05-25 |
| 群聊封禁 | 严格审核群成员，禁止用户发布内容，运营发布为主 | 运营 | 持续 |
| 个人号限制 | 控制添加好友频率 (每日<50人)，使用多号轮换 | 运营 | 持续 |
| 支付限制 | 不依赖个人收款码，使用小报童/知识星球正规支付渠道 | 产品 | 2026-05-18 |

### 5.4 微信导流最佳实践

```
安全导流路径:
1. 公众号文章 → 运营程序/企业微信联系 → 个人号深度沟通 → 付费转化
2. 群聊仅作为内容分发和价值交付，不作为售后服务主渠道
3. 付费通过小报童/知识星球/公众号内支付，不使用个人收款码
```

## 六小红书风险评估

### 6.1 风险总览

| 风险项 | 等级 | 概率 | 影响 | 证据来源 |
|---------|------|------|------|----------|
| 账号封禁 | **严重** | **高** | 完全丧失获客渠道 | 小红书社区案例 |
| 内容限流 | **高** | 高 | 曝光量严重下降 | 算法调整 |
| 引流扣分 | **高** | 高 | 笔记/个人简介中的联系方式被屏蔽 | 平台审核 |
| 评论区封控 | **高** | 高 | 无法通过评论引导私信 | 风控系统 |
| 商业化限制 | **中** | 高 | 不允许直接导流到外部订阅 | 平台规则 |

### 6.2 小红书封号高危行为

```根据小红书社区规范和用户反馈整理:

高危行为 (导致封禁):
- 在笔记/评论/个人简介中直接放微信号/公众号/外部链接
- 频繁发送类似"加微信领资料"等导流话术
- 发布"赚钱"、"副业"、"开店"等敏感话题
- 账号行为模式类似营销号 (只发布导流内容)
- 被多人举报

中危行为 (导致限流):
- 发布跟风/复制内容
- 标题党/失实宣传
- 评论区互动频率异常
```

### 6.3 执行风控策略

| 风险 | 缓解措施 | 责任人 | 截止日期 |
|------|----------|----------|----------|
| 帐号封禁 | **不作为自动发布渠道**; 如果使用，严格遵守内容合规清单 | 运营 | 2026-05-18 |
| 内容限流 | 优化内容质量，避免跟风/复制 | 运营 | 每篇发布前 |
| 引流扣分 | 使用口令/暗号/图片隐藏方式，避免直接联系方式 | 运营 | 每篇发布前 |
| 商业化限制 | 小红书仅作为品牌曝光，不作为直接转化渠道 | 产品 | 2026-05-18 |

### 6.4 小红书使用建议

```
不推荐小红书作为自动发布渠道的原因:
1. 封号率极高，且不可预测
2. 引流效率低，平台对商业化内容限制严格
3. 内容监管严，"赚钱"类话题容易触发审核
4. 运营成本高，需要持续账号养护和备用号准备

如果硬要使用小红书:
- 使用个人账号，不使用企业号 (企业号审核更严)
- 内容主打价值分享，不主打营销
- 导流使用口令 (如"回复'资料'获取更多")
- 准备多个备用账号
- 不要在简介中直接放联系方式
```

## 九备选方案汇总

### 9.1 若 Substack/Stripe 不可用

```
备选方案A: 小报童 (推荐)
- 优点: 支持中国大陆支付，上手快，抽成比例合理
- 缺点: 平台依赖，用户数据不完全掌握
- 适用场景: 目标用户主要在中国大陆

备选方案B: 知识星球
- 优点: 社群功能强，适合高价值订阅
- 缺点: 抽成较高，社区质量参差不齐
- 适用场景: 需要强社群运营的高价产品

备选方案C: 自建支付 (不推荐早期使用)
- 优点: 完全掌握用户和收入
- 缺点: 开发成本高，合规风险
- 适用场景: 月收入稳定超过¥50,000后考虑
```

### 9.2 若微信私域受限

```
备选渠道:
1. 即刾App (批量导流到邮箱订阅)
2. 知乎 (长文+回答导流)
3. V2EX/独立开发者社区
4. Twitter/X (英文用户)
```
