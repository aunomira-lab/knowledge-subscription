## 一、目标

建立可复制的每日运营流程，确保内容生产、发布、收款、客服、数据记录形成闭环，不遗漏任何赚钱机会。

---

## 二、每日运营节奏（SOP）

### 08:00 — 生成日报

| 步骤 | 动作 | 命令/工具 | 验收标准 |
|------|------|-----------|---------|
| 1 | 运行日报生成器 | `python app/generate_substack_issue.py` | 文件 `reports/daily/YYYY-MM-DD.md` 存在 |
| 2 | 质量检查 | `python scripts/validate_monitoring_system.py` | 质量分 >= 80% |
| 3 | 人工审核 | 阅读全文，检查事实错误 | 无明显错误，标题有钩子 |
| 4 | 定稿 | 保存为 `reports/daily/YYYY-MM-DD.md` | 文件大小 > 2KB |

**质量红线**：任何含事实错误、无明确行动建议、无钩子的内容不得发布。违规 = 扣减当日绩效。

### 10:00 — 多平台发布

| 平台 | 内容形式 | 发布频率 | 引流动作 | 负责人 |
|------|----------|---------|---------|--------|
| 即刻 | 每日头条 + 短评 | 每日1条 | 文末带"订阅完整版" | dev-monitor |
| 知乎 | 长文/回答 | 每日1篇 | 个人简介+公众号 | dev-monitor |
| 小红书 | 图文笔记 | 每日1篇 | 评论区置顶收款码 | dev-monitor |
| 微信公众号 | 深度文章 | 每日1篇 | 菜单栏+自动回复 | dev-monitor |
| GitHub | README更新 | 每周2次 | Sponsor按钮+订阅链接 | dev-coder |

**发布 checklist**：
- [ ] 标题含数字或痛点词
- [ ] 前3句有钩子
- [ ] 文末有明确的CTA（行动号召）
- [ ] 包含收款码/订阅链接
- [ ] 已保存发布截图

### 15:00 — 收款与记录

| 步骤 | 动作 | 验收标准 |
|------|------|---------|
| 1 | 检查微信/支付宝收款记录 | 截图保存到 `receipts/YYYY-MM-DD/` |
| 2 | 记录到 CSV | 更新 `metrics/experiment_tracker.csv` 当日行 |
| 3 | 用户确认 | 发送欢迎消息，拉入付费群 |
| 4 | 备份 | 收款截图上传云盘 |

**收款记录字段**：
- 时间戳
- 金额
- 渠道（微信/支付宝/小报童/爱发电）
- 用户标识（去敏化，如 WX-001）
- 套餐类型（早鸟/标准/专业/定制）
- 截图文件名

### 17:00 — 用户互动

| 动作 | 目标 | 时间上限 |
|------|------|---------|
| 回复评论/私信 | 100%回复率 | 30分钟 |
| 高意向用户1v1 | 转化免费→付费 | 30分钟 |
| 社群维护 | 保持活跃 | 15分钟 |

### 21:00 — 数据更新与看板

| 步骤 | 动作 | 命令 |
|------|------|------|
| 1 | 更新 experiment_tracker.csv | 手动编辑或运行脚本 |
| 2 | 生成看板 | `python scripts/monitor_dashboard.py` |
| 3 | 检查预警 | 阅读脚本输出 |
| 4 | 填写日报 | `docs/daily_log/YYYY-MM-DD.md` |

### 22:00 — 备份

| 备份项 | 位置 | 方式 |
|--------|------|------|
| 日报内容 | `reports/daily/` | git commit |
| 收款截图 | `receipts/` | 云盘同步 |
| 用户数据 | `metrics/experiment_tracker.csv` | git commit + 云盘 |
| 看板报告 | `reports/daily/DASHBOARD_*.md` | git commit |

## 三、内容质量门禁

### 准入标准

| 维度 | 标准 | 检查方式 |
|------|------|---------|
| 事实准确性 | 数据/引用可溯源 | 搜索引擎验证 |
| 行动价值 | 读者能立即执行至少1个动作 | 人工判断 |
| 钩子强度 | 标题打开率目标>5% | A/B测试 |
| 视觉规范 | 排版清晰，无大段文字 | 人工检查 |
| 合规性 | 无敏感词、无侵权 | 敏感词检查 |

### 发布前必做

```bash
# 质量检查脚本
python scripts/validate_monitoring_system.py

# 如果失败，禁止发布
if [ $? -ne 0 ]; then
    echo "质量门禁未通过，禁止发布"
    exit 1
fi
```

## 四、用户入口管理

### 免费用户入口

| 入口 | 路径 | 响应 |
|------|------|------|
| 公众号回复"试读" | 自动回复 free_preview.md | 发送免费试读包 |
| 即刻/知乎/小红书私信 | 手动回复 | 发送公众号二维码 |
| GitHub Sponsor | README 链接 | 引导至订阅页 |

### 付费用户入口

| 入口 | 路径 | 响应 |
|------|------|------|
| 扫码支付 | 微信/支付宝收款码 | 发送确认消息+拉群 |
| 小报童 | 平台内订阅 | 自动开通权限 |
| 爱发电 | 平台内订阅 | 自动开通权限 |

### 用户分层

```
[曝光] 平台浏览
    └── [免费] 获取试读版
            ├── [付费] 扫码支付 → 付费群+日报推送
            └── [流失] 7天未转化 → 发问卷调查
```

## 六、异常处理

| 异常 | 处理流程 | 升级路径 |
|------|---------|---------|
| 内容生成失败 | 启用备用模板 → 手动编辑 → 延迟发布 | 2小时内未解决 → 暂停当日发布 |
| 收款码异常 | 切换备用渠道 → 通知用户 → 修复 | 30分钟内未解决 → 暂停收款 |
| 平台限流/封禁 | 切换渠道 → 申诉 → 备份内容 | 立即升级至 dev-deploy |
| 用户投诉 | 记录 → 道歉 → 退款/补偿 → 复盘 | 3次以上投诉 → 升级至 dev-security |
| 竞品抄袭 | 截图取证 → 保留法律权利 → 差异化 | 法律风险 → 升级至 dev-security |
