## 一、支持理念

- **目标**: 在4小时内解决普通咨询，24小时内响应所有问题
- **原则**: 先对走钱，再对人（退款/投诉优先级最高）
- **风格**: 专业、友好、不过度承诺
- **语言**: 中文为主，英文用户可用简单英文

## 三、问题分类与处理流程

### 类别A：付费与订单问题（级别=P1）

**包含场景**: 无法支付、重复扣款、退款请求、不清楚定价

**处理步骤**:
1. 立即确认收到反馈（自动回复模板）
2. 检查对应订单状态
3. 若是支付故障：测试支付流程→提供备用支付链接或微信收款码
4. 若是退款：
   - 购买不满足7天：无条件全额退款
   - 购买超7天：根据使用情况部分退款（不得少于50%）
   - 有投诉：无条件全额退款
5. 记录到客服记录表
6. 2h内给出解决方案

**标准回复模板**:
```
亲爱的订阅老铁，

已收到您的问题反馈。

问题类型: {category}
当前状态: 正在处理中
预计解决时间: {time}

我们将在{time}内给您具体回复。如需加急请发消息说明。

感谢支持！
```

### 类别B：内容访问问题（级别=P2）

**包含场景**: 收不到邮件、链接失效、平台登录不上、内容与描述不符

**处理步骤**:
1. 确认收到反馈
2. 检查用户订阅状态
3. 重发邮件/重置链接/重设登录凭证
4. 若是内容质量问题：记录具体批评点，转交内容团队
5. 4h内给出解决方案

### 类别C：功能建议与广泛咨询（级别=P3）

**包含场景**: 新功能建议、流量合作、媒体采访、约稿定制服务

**处理步骤**:
1. 记录建议/需求
2. 给出正式回复
3. 功能建议纳入产品迭代计划
4. 定制服务直接引导至定件派单
5. 24h内给出回复

### 类别D：绕过收款订单问题（级别=P1）

**包含场景**: 用户扫码支付后没有收到激活链接、重复扣款、订单金额异常

**处理步骤**:
1. 立即确认微信/支付宝收款记录
2. 核实支付金额和时间
3. 手动开通权限并发送激活链接
4. 记录到 experiment_tracker.csv 的绕过收款栏位
5. 30分钟内给出解决方案

| 级别 | 响应时限 | 解决时限 | 升级条件 |

## 五、常见问题快速处理表

| 常见问题 | 判断条件 | 解决方案 | 参考时间 |
|----------|----------|----------|----------|
| 收不到邮件 | 测试发送失败 | 检查垃圾邮箱→添加白名单→重发 | 5分钟 |
| 链接打不开 | 网页返回非200 | 检查链接有效期→更换平台镜像→重发 | 10分钟 |
| 支付失败 | 用户截图显示错误 | 切换备用支付链接→手动发起微信转账→人工开通权限 | 15分钟 |
| 想退款 | 购买不满→第1天 | 确认订单→微信/支付宝原路退款→发送确认消息 | 10分钟 |
| 想改定价 | 用户对定价有疑问 | 说明当前定价方案→提供比价表→如适用给专属优惠 | 10分钟 |
| 内容质量投诉 | 用户明确表达不满意 | 说明内容策略→记录具体投诉点→承诺优化→退款保底 | 15分钟 |
| 如何定制报告 | 用户询问定制服务 | 发送定制服务派单→明确报价¥499→约时间沟通需求 | 10分钟 |
| 平台与我们的区别 | 用户比价比货 | 发送对比表→强调独立开发者持续迭代→免费试看一篇 | 5分钟 |
| 绕过收款后未开通权限 | 用户扫码支付后没收到内容 | 核实支付记录→手动开通→发送激活链接→记录到CSV | 10分钟 |

## 六、客服记录表

每次客服交互必须记录（在每日更新 experiment_tracker.csv 时同步更新）:

| 字段 | 说明 |
|------|------|
| 日期 | 交互日期 |
| 用户ID | 匿私化ID（如微信前3后2或邮箱前后） |
| 渠道 | 微信/邮件/小报童/知乎/小红书 |
| 类别 | A付费/B内容/C建议/D绕过收款 |
| 级别 | P1/P2/P3/P4 |
| 问题摘要 | 20字以内概括 |
| 处理方案 | 简要说明 |
| 解决时间 | 实际耗时 |
| 结果 | 已解决/待跟进/已升级 |
| 收入影响 | 是否涉及退款、赔偿、促单 |

## 七、升级路径

### 当涉及以下情况时，必须升级给小电（参见 incident_runbook.md）

1. 单日投诉≥3件或退款≥2件
2. 支付渠道被平台封禁
3. 内容被平台删除或限流
4. 用户数据泄露风险
5. 涉及法律诉讼或平台处罚
6. 绕过收款码被平台封禁（微信/支付宝）

**升级方式**: 微信通知小电 + 在 project_state.json 标记紧急状态
