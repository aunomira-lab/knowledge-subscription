## 一清单使用说明

### 使用方法

```检查流程:
1. 内容创作完成后，创作者先自检
2. 发布前由负责人复核
3. 在内容管理表中记录检查结果
4. 发现问题立即修改，不通过不发布

记录方式:
- 通过: ✓
- 不通过: ✗ (需注明原因)
- 不适用: N/A
```

### 责任分工

| 角色 | 职责 | 验证方式 |
|------|------|----------|
| 创作者 | 完成初稿，自检清单 | 签名确认 |
| 编辑 | 复核内容，执行清单 | 签名确认 |
| 安全审核 | 抽查高风险内容 | 签名确认 |

## 二通用合规检查 (所有平台)

### 2.1 非法内容检查 (绝对禁区)

| 检查项 | 检查要点 | 通过 | 备注 |
|---------|----------|-------|-------|
| 政治敏感内容 | 不涉及政治、政党、政府评论 | [ ] | 包括显性和隐性表达 |
| 色情内容 | 无色情、淫秽、暴露内容 | [ ] | 包括文字、图片、外链 |
| 暴力内容 | 无暴力、恐怖、自残内容 | [ ] | |
| 贩毒/赌博 | 不涉及毒品、赌博推广 | [ ] | 包括"探索"类表述 |
| 欺诈信息 | 无欺诈、传销、迷信内容 | [ ] | |
| 侵权内容 | 不侵犯他人隐私/商业秘密 | [ ] | |
| 作品版权 | 引用内容已授权或合理使用 | [ ] | 注明来源和授权 |

```bash

### 2.2 过度承诺检查

| 检查项 | 危险表述 | 安全做法 | 通过 |
|---------|----------|----------|-------|

## 三平台特定合规检查

### 3.1 微信公众号合规检查

#### 3.1.1 运营规范

| 检查项 | 规范要求 | 通过 | 备注 |
|---------|----------|-------|-------|
| ICP备案 | 已完成ICP备案并正确显示 | [ ] | 资讯类公众号必须 |
| 发布频率 | 每日不超过1篇，无弹幕推送 | [ ] | 避免骚扰用户 |
| 广告比例 | 广告内容 < 20% | [ ] | 以价值内容为主 |
| 外链管理 | 外部链接经过审核 | [ ] | 避免恶意网站 |

#### 3.1.2 导流规范

| 检查项 | 禁止行为 | 安全做法 | 通过 |
|---------|----------|----------|-------|
| 二维码 | 文章内直接放个人二维码 | 使用运营程序引导 | [ ] |
| 外部链接 | 放非白名单外部链接 | 使用公众号内页面 | [ ] |
| 导流话术 | "加我微信"、"扫码" | "回复XX获取"、"点击了解更多" | [ ] |
| 诱导分享 | "转发到群抽奖" | 自然分享鼓励 | [ ] |

#### 3.1.3 敏感词清单 (微信特有)

```
高危词 (绝对禁用):
- 加我微信 / 加微 / VX / 微信号
- 扫码 / 二维码
- 免费领 / 领红包
- 转发抽奖 / 分享有礼
- 点赞 / 在看 / 点在看

中危词 (尽量避免):
- 赚钱 (可用"变现"替换)
- 副业 (可用"第二曲线"替换)
- 投资 (可用"投入"替换)
- 最 / 第一 / 绝对

可用替代词:
- 赚钱 → 变现 / 盈利 / 收益 / 回报
- 副业 → 第二曲线 / 创业 / 探索
- 加微信 → 联系 / 沟通 / 交流
```

### 3.2 小红书合规检查

#### 3.2.1 内容限制

#### 3.2.2 小红书敏感词清单

```
绝对禁用 (导致封号):
- 赚钱 / 快速赚钱 / 轻松赚钱
- 副业探索 / 副业必看
- 日入 / 月入 / 年入
- 零成本创业 / 零投入赚钱
- 加我 / 加微信 / 私信我
- 扫码 / 二维码
- 链接在简介 / 点我主页

尽量避免 (可能限流):
- 开店 / 创业 / 投资
- 财富自由 / 财务自由
- 稳赚 / 保证 / 肯定

安全替代:
- 赚钱 → 收益 / 回报 / 成长
- 副业 → 兴趣变现 / 技能变现
- 日入 → 日常收益 / 每日进盈

## 五风险应对预案

### 5.1 内容被删/限流应急预案

```级别响应:

红色 (内容被删或账号封禁):
1. 立即停止所有该平台发布
2. 检查被删内容，确定违规点
3. 修改内容后重新提交审核
4. 启动备用渠道
5. 记录事件，更新合规清单

黄色 (内容被限流):
1. 分析限流原因
2. 调整内容策略
3. 减少该类内容比例
4. 监控后续表现
```

### 5.2 用户举报应对

```应对流程:
1. 收到举报通知后2小时内响应
2. 审查被举报内容
3. 确认是否违规
4. 如违规，立即删除并道歉
5. 如未违规，申诉并准备证据
6. 记录事件，更新合规清单
```

## 附录: 常用检查工具

### A. 敏感词检测脚本

```python
#!/usr/bin/env python3
# check_sensitive_words.py

SENSITIVE_WORDS = {
    "所有平台": [
        "政治", "党", "色情", "淫秽", "暴力", "恐怖",
        "毒品", "赌博", "传销", "欺诈"
    ],
    "微信公众号": [
        "加我微信", "加微", "VX", "扫码",
        "免费领", "转发抽奖", "点赞", "在看"
    ],
    "小红书": [
        "赚钱", "副业", "日入", "月入", "年入",
        "零成本创业", "加我", "私信我", "扫码"
    ],
    "短视频": [
        "刷粉", "代运营", "赚快钱", "加群", "拉群"
    ]
}

def check_content(content, platform="所有平台"):
    """检查内容中的敏感词"""
    words = SENSITIVE_WORDS.get(platform, [])
    found = []
    for word in words:
        if word in content:
            found.append(word)
    return found

if __name__ == "__main__":
    import sys
    if len(sys.argv) < 2:
        print("用法: python check_sensitive_words.py <文件路径> [平台]")
        sys.exit(1)
    
    filepath = sys.argv[1]
    platform = sys.argv[2] if len(sys.argv) > 2 else "所有平台"
    
    with open(filepath, 'r', encoding='utf-8') as f:
        content = f.read()
    
    found = check_content(content, platform)
    
    if found:
        print(f"⚠️ 发现敏感词 ({platform}): {', '.join(found)}")
        sys.exit(1)
    else:
        print(f"✓ 未发现敏感词 ({platform})")
        sys.exit(0)
```
