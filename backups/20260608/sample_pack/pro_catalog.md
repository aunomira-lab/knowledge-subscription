## 一、内容模块清单

### 模块A: 每日机会雷达 (核心交付)

**频率**: 每日1期（工作日），周末总结  
**形式**: Markdown + 可选的PDF导出  
**内容结构**:

```
每期包含:
├── 头条机会 (1条深度分析)
│   ├── 机会背景与数据支撑
│   ├── 竞品分析 (3-5个竞品)
│   ├── 执行步骤拆解
│   ├── 所需资源清单
│   └── 预期收益与风险评估
├── 快速机会 (2-3条简报)
│   ├── 一句话总结
│   ├── 数据来源
│   └── 行动建议
└── 数据更新
    ├── 监控指标变化
    └── 趋势预警
```

**示例**: 见 `week1_samples/day1_opportunity_radar.md`

---

### 模块B: 可执行资源包

**频率**: 每周3-5个  
**内容类型**:

| 资源类型 | 说明 | 示例 |
|---------|------|------|
| Python脚本 | 数据采集、自动化任务 | 网站监控爬虫、RSS聚合器 |
| n8n工作流 | 现成的工作流模板 | 小红书自动发布、邮件自动化 |
| Prompt模板 | 经过测试的AI提示词 | 内容生成、数据分析 |
| 检查清单 | 执行某机会的标准流程 | 产品上线 checklist |
| 代码片段 | 即插即用的代码 | API调用、数据处理 |

**交付位置**: `resources/YYYY-MM/`  
**示例**: 见 `week1_samples/resources/`

---

**访问方式**: 静态HTML页面 / Notion数据库  
**示例**: 见 `app/dashboard.html`

## 四、技术交付规格

### 4.1 文件格式

| 内容 | 主要格式 | 辅助格式 |
|------|---------|---------|
| 日报 | Markdown | PDF, HTML |
| 脚本 | Python (.py) | Jupyter Notebook |
| 工作流 | n8n JSON | 截图+说明 |
| 数据 | CSV/JSON | 可视化图表 |

### 4.2 文件命名规范

```
reports/
├── YYYY-MM/
│   ├── issue-XXX-topic-name.md      # 日报
│   └── week-X-summary.md            # 周报
├── sample_pack/                      # 样例包
└── resources/
    └── YYYY-MM/
        ├── n8n-workflow-name.json   # n8n工作流
        ├── script-name.py            # Python脚本
        └── prompt-name.md            # Prompt模板
```

### 4.3 代码交付标准

- 必须包含 README.md 说明
- 必须包含 requirements.txt 或依赖清单
- 必须有示例运行命令
- 必须有预期输出示例
- 代码需通过基本语法检查
